# Theming

The Iqvizyon UI Theme is represented by a set of tokens. Each token resolves to a single value which can be assigned to a CSS property.

```tsx
const exampleTheme = {
  borderRadiusSmall: '2px',
  //...
  colorNeutralForeground2: '#424242',
};
```

You can browse all the available tokens in **[Theme](/docs/theme-colors--docs)** section of the docs.

## How theme is applied

No matter what theme is used, the component styles are always the same. The only way to change the component styling is through theme tokens which can be used in style values.

Those tokens are resolved to CSS variables. The `IqvizyonProvider` component is responsible for setting the values of the CSS variables in DOM and changing them when the theme changes. When the theme is switched, only the variables are changed, all styles remain the same.

Place a `<IqvizyonProvider />` at the root of your app and pass a theme to the `theme` prop. The provider will render a `div` and set all tokens as CSS variables on that element. The provider also propagates CSS variables to React portals created with [Portal component](/docs/components-portal-portal--docs).

```jsx
import { IqvizyonProvider, teamsLightTheme } from '@iqvizyonui/react-components';

export const AppRoot = ({children}) => (
  <IqvizyonProvider theme={teamsLightTheme}>
    {children}
  </IqvizyonProvider>,
);
```

Import `tokens` to style a component using `makeStyles`

```jsx
import { tokens } from '@iqvizyonui/react-components';

const useStyles = makeStyles({
  root: { display: 'flex' },
  rootPrimary: { color: tokens.colorNeutralForeground3 },
});

export Component = props => {
  const classes = useStyles();

  return <div className={mergeClasses('ui-component', classes.root, props.primary && classes.rootPrimary)} />;
}
```

For more details, see [Styling components](?path=/docs/concepts-developer-styling-components--docs).

### Do not use CSS variables directly

**⚠ Never use theme CSS variables directly!** The CSS variables implementation of the theme is internal to the library. We might eventually decide to change the variable names, hash them or even use direct values instead of some variables. Always use the `tokens` to access the theme.

## Available themes

Iqvizyon UI currently exports following themes:

- Web Light (`webLightTheme`)
- Web Dark (`webDarkTheme`)
- IQV Light (`teamsLightTheme`)
- IQV Dark (`teamsDarkTheme`)
- IQV High Contrast (`teamsHighContrastTheme`)

### High contrast themes

⚠ Do not use High Contrast themes! All Iqvizyon UI components support Windows High Contrast mode automatically regardless of the active theme.
Windows high contrast mode is the recommended high contrast platform for all customers using Iqvizyon UI.

Hardcoded High Contrast themes are considered legacy, to be used only in applications which explicitly support those.

## Customizing the theme

Applications can customize a theme in multiple ways.

### Custom Brand ramp

The brand ramp is a color ramp going from dark to light colors:

<img className="sbdocs-img" src={require('../../public/brand-ramp-example.png')} alt="Example of a brand ramp" />

A theme is derived from a brand ramp. To use a theme with a custom brand ramp, instead of importing a predefined theme, you can use theme factory functions.

The following factory functions are available:

- `createLightTheme()`
- `createDarkTheme()`
- `createTeamsDarkTheme()`
- `createHighContrastTheme()`

```tsx
import { BrandVariants, createLightTheme, createDarkTheme } from '@iqvizyonui/react-components';

const customBrandRamp: BrandVariants = {
  10: '#008',
  //...
  160: '#88F',
};

export const customLightTheme = createLightTheme(customBrandRamp);
export const customDarkTheme = createDarkTheme(customBrandRamp);
```

### Overriding existing tokens

⚠️ If the existing tokens do not fulfill your needs, you should talk to your designer instead of overriding tokens.

A theme is a flat object containing `{ [token name]: CSS value }` pairs. You can copy the object and overwrite any tokens you wish.

```tsx
import { webLightTheme, Theme } from '@iqvizyonui/react-components';

export const customLightTheme: Theme = {
  ...webLightTheme,
  colorNeutralForeground1: '#555', // overriden token
};
```

### Extending theme with new tokens

It's often useful for an app to extend the base set of tokens from Iqvizyon UI. This process will help consuming teams or libraries add more tokens, but sharing them is outside the scope of this doc.

⚠ Warning that adding more tokens adds more CSS variables which can effect run time performance as each DOM Node carries all the tokens.

```tsx
import { makeStyles, themeToTokensObject, webLightTheme, IqvizyonProvider, Theme } from '@iqvizyonui/react-components';

// You can pass your own custom tokens to a theme and pass that to the provider.
type CustomTheme = Theme & {
  tokenA: string;
  tokenB: string;
  tokenC: string;
};
const customTheme: CustomTheme = { ...webLightTheme, tokenA: 'red', tokenB: 'blue', tokenC: 'green' };
function App() {
  return <IqvizyonProvider theme={customTheme}>{/* ... */}</IqvizyonProvider>;
}

// ...

// You can construct a custom tokens object by yourself.
const customTokens: Record<keyof CustomTheme, string> = {
  ...tokens,
  tokenA: `var(--tokenA)`,
  tokenB: `var(--tokenB)`,
  tokenC: `var(--tokenC)`,
};

// You can alternatively use the themeToTokensObject function to construct the custom tokens object.
// Note: If you do it via the themeToTokensObject you might see a negative effect on tree-shaking since bundles won't know the shape of the output.
const alternativeCustomTokens = themeToTokensObject(customTheme);

// You can then use this custom tokens object inside your styles.
const useStyles = makeStyles({
  base: {
    color: customTokens.tokenA,
    backgroundColor: customTokens.tokenB,
    outlineColor: customTokens.tokenC,
  },
});
```

# Temalandırma

Iqvizyon UI teması bir belirteç kümesiyle temsil edilir. Her belirteç, bir CSS özelliğine atanabilecek tek bir değere çözümlenir.

```tsx
const exampleTheme = {
  borderRadiusSmall: '2px',
  colorNeutralForeground2: '#424242',
};
```

Kullanılabilir belirteçlerin tümünü belgelerin **[Tema](/docs/theme-colors--docs)** bölümünde inceleyebilirsiniz.

## Tema Nasıl Uygulanır?

Bileşen stilleri kullanılan temadan bağımsız olarak aynı kalır. Bileşen görünümünü değiştirmek için stil değerlerinde tema belirteçlerini kullanın.

Belirteçler CSS değişkenlerine çözümlenir. `IqvizyonProvider`, DOM üzerindeki CSS değişkenlerini ayarlar ve tema değiştiğinde değerlerini günceller. Tema değiştirildiğinde stiller değil, yalnızca değişkenler değişir.

Uygulamanızın köküne bir `<IqvizyonProvider />` yerleştirin ve `theme` prop'una bir tema verin. Sağlayıcı bir `div` oluşturur, belirteçleri bu öğe üzerinde CSS değişkenleri olarak ayarlar ve değişkenleri [Portal bileşeni](/docs/components-portal-portal--docs) ile oluşturulan React portallarına iletir.

```jsx
import { IqvizyonProvider, teamsLightTheme } from '@iqvizyonui/react-components';

export const AppRoot = ({ children }) => <IqvizyonProvider theme={teamsLightTheme}>{children}</IqvizyonProvider>;
```

Bir bileşeni `makeStyles` ile stillendirmek için `tokens` içe aktarın:

```jsx
import { tokens } from '@iqvizyonui/react-components';

const useStyles = makeStyles({
  root: { display: 'flex' },
  rootPrimary: { color: tokens.colorNeutralForeground3 },
});

export const Component = props => {
  const classes = useStyles();
  return <div className={mergeClasses('ui-component', classes.root, props.primary && classes.rootPrimary)} />;
};
```

Ayrıntılar için [Bileşenleri stillendirme](?path=/docs/concepts-developer-styling-components--docs) bölümüne bakın.

### CSS Değişkenlerini Doğrudan Kullanmayın

**⚠ Tema CSS değişkenlerini doğrudan kullanmayın.** CSS değişkeni uygulaması kütüphanenin iç ayrıntısıdır ve gelecekte değişebilir. Temaya erişmek için her zaman `tokens` kullanın.

## Kullanılabilir Temalar

Iqvizyon UI şu temaları dışa aktarır:

- Web Açık (`webLightTheme`)
- Web Koyu (`webDarkTheme`)
- IQV Açık (`teamsLightTheme`)
- IQV Koyu (`teamsDarkTheme`)
- IQV Yüksek Karşıtlık (`teamsHighContrastTheme`)

### Yüksek Karşıtlık Temaları

⚠ Yüksek Karşıtlık temalarını kullanmayın. Iqvizyon UI bileşenleri, etkin temadan bağımsız olarak Windows Yüksek Karşıtlık modunu otomatik biçimde destekler. Doğrudan kodlanan Yüksek Karşıtlık temaları eski kabul edilir ve yalnızca bunları açıkça destekleyen uygulamalarda kullanılmalıdır.

## Temayı Özelleştirme

Uygulamalar bir temayı birkaç şekilde özelleştirebilir.

### Özel Marka Skalası

Marka skalası koyudan açığa ilerleyen bir renk skalasıdır:

<img className="sbdocs-img" src={require('../../public/brand-ramp-example.png')} alt="Marka skalası örneği" />

Bir tema marka skalasından türetilir. Özel bir marka skalası kullanmak için hazır bir temayı içe aktarmak yerine tema fabrika işlevlerini kullanın:

- `createLightTheme()`
- `createDarkTheme()`
- `createTeamsDarkTheme()`
- `createHighContrastTheme()`

```tsx
import { BrandVariants, createLightTheme, createDarkTheme } from '@iqvizyonui/react-components';

const customBrandRamp: BrandVariants = {
  10: '#008',
  160: '#88F',
};

export const customLightTheme = createLightTheme(customBrandRamp);
export const customDarkTheme = createDarkTheme(customBrandRamp);
```

### Mevcut Belirteçleri Geçersiz Kılma

⚠️ Mevcut belirteçler gereksinimlerinizi karşılamıyorsa belirteçleri geçersiz kılmadan önce tasarımcınızla görüşün.

Tema, `{ [belirteç adı]: CSS değeri }` çiftlerini içeren düz bir nesnedir. Nesneyi kopyalayıp gereken belirteçleri geçersiz kılabilirsiniz.

```tsx
import { webLightTheme, Theme } from '@iqvizyonui/react-components';

export const customLightTheme: Theme = {
  ...webLightTheme,
  colorNeutralForeground1: '#555',
};
```

### Temayı Yeni Belirteçlerle Genişletme

Bir uygulama, Iqvizyon UI temel belirteç kümesini kendi belirteçleriyle genişletebilir. Daha fazla belirteç, her DOM düğümüne daha fazla CSS değişkeni eklediği için çalışma zamanı performansını etkileyebilir.

```tsx
type CustomTheme = Theme & {
  tokenA: string;
  tokenB: string;
  tokenC: string;
};

const customTheme: CustomTheme = {
  ...webLightTheme,
  tokenA: 'red',
  tokenB: 'blue',
  tokenC: 'green',
};

function App() {
  return <IqvizyonProvider theme={customTheme}>{/* ... */}</IqvizyonProvider>;
}
```

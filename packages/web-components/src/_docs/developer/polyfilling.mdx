## HTML Popover

From Iqvizyon UI Web Components V3 forward, popover-like components will make use of the newer [HTML popover attribute](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/popover). Broader `[popover]` support beyond the browser baseline will require a polyfill.

To get started, add the [popover polyfill](https://github.com/oddbird/popover-polyfill/) to your package.

```bash
yarn install @oddbird/popover-polyfill
```

Then in your `main.js` file after your `setTheme()` call, check for `popover` support and lazily import the package if not supported.

```js
(async () => {
  if (!('popover' in HTMLElement.prototype)) {
    await import('@oddbird/popover-polyfill');
  }
})();
```

Two lines of global CSS are needed to cleanup a potential positioning side effect created by the polyfill and LightDOM authored child components.

```css
[popover].\:popover-open {
  inset: unset;
  border: 1px solid transparent;
}
```

## CSS Anchor Positioning

Menu anchor positioning is handled minimally in a fallback script but for more complex anchor positioned components (Tooltip, Dropdown, Combobox), you will need to implement your own polyfill.

```js
/**
 * Client-side Import example with conditional polyfill import for older browsers.
 * This MUST be included before Iqvizyon UI.
 */
if (!CSS.supports('anchor-name: --foo')) {
  const { default: applyPolyfill } = await import(
    'https://unpkg.com/@oddbird/css-anchor-positioning/dist/css-anchor-positioning-fn.js'
  );
  window.CSS_ANCHOR_POLYFILL = applyPolyfill;
}
```

```js
/**
 * NPM Import example where polyfill is bundled into main bundle
 * This MUST be included before Iqvizyon UI.
 */
import { default as applyPolyfill } from '@oddbird/css-anchor-positioning/fn';
window.CSS_ANCHOR_POLYFILL = applyPolyfill;
```

## HTML Focusgroup

For components that require directional navigations (moving focus between focusable elements within a component with arrow keys instead of tab), Iqvizyon Web Components have adopted [HTML Focusgroup](https://open-ui.org/components/scoped-focusgroup.explainer/). The components that currently use focusgroup include MenuList/MenuItem, RadioGroup/Radio, Tablist/Tab, and Tree/TreeItem.

For browsers that don’t yet support focusgroup, we use [`@microsoft/focusgroup-polyfill`](https://github.com/microsoft/polyfills/tree/main/packages/focusgroup) and automatically polyfill the components when they are connected to the DOM.

If you want to opt out of the polyfill, you can use the base classes of these components, e.g.

```js
import { BaseTablist } from '@iqvizyonui/web-components/tablist/base.js';

export class MyTablist extends BaseTablist {}
```

## HTML Popover

Iqvizyon UI Web Components V3 ve sonraki sürümlerde, popover benzeri bileşenler yeni [HTML popover özniteliğinden](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/popover) yararlanır. Tarayıcı temel düzeyinin ötesinde daha geniş `[popover]` desteği için polyfill gerekir.

Başlamak için paketinize [popover polyfill](https://github.com/oddbird/popover-polyfill/) ekleyin.

```bash
yarn install @oddbird/popover-polyfill
```

Ardından `main.js` dosyanızda, `setTheme()` çağrınızdan sonra `popover` desteğini denetleyin ve desteklenmiyorsa paketi gecikmeli olarak içe aktarın.

```js
(async () => {
  if (!('popover' in HTMLElement.prototype)) {
    await import('@oddbird/popover-polyfill');
  }
})();
```

Polyfill ve LightDOM tarafından oluşturulan alt bileşenlerin neden olabileceği olası konumlandırma yan etkisini temizlemek için iki satır genel CSS gerekir.

```css
[popover].\:popover-open {
  inset: unset;
  border: 1px solid transparent;
}
```

## CSS Anchor Positioning

Menü ankraj konumlandırması bir geri dönüş betiğiyle temel düzeyde ele alınır. Ancak daha karmaşık ankraj konumlu bileşenler (`Tooltip`, `Dropdown`, `Combobox`) için kendi polyfill’inizi uygulamanız gerekir.

```js
/**
 * Eski tarayıcılar için koşullu polyfill içe aktarması kullanan istemci tarafı içe aktarma örneği.
 * Bu kod Iqvizyon UI’den önce eklenmelidir.
 */
if (!CSS.supports('anchor-name: --foo')) {
  const { default: applyPolyfill } = await import(
    'https://unpkg.com/@oddbird/css-anchor-positioning/dist/css-anchor-positioning-fn.js'
  );
  window.CSS_ANCHOR_POLYFILL = applyPolyfill;
}
```

```js
/**
 * Polyfill’in ana pakete dahil edildiği NPM içe aktarma örneği.
 * Bu kod Iqvizyon UI’den önce eklenmelidir.
 */
import { default as applyPolyfill } from '@oddbird/css-anchor-positioning/fn';
window.CSS_ANCHOR_POLYFILL = applyPolyfill;
```

## HTML Focusgroup

Yönlü gezinme gerektiren bileşenlerde, örneğin sekme tuşu yerine ok tuşlarıyla bir bileşen içindeki odaklanılabilir öğeler arasında geçiş için, Iqvizyon Web Components [HTML Focusgroup](https://open-ui.org/components/scoped-focusgroup.explainer/) özelliğini benimser. Şu anda focusgroup kullanan bileşenler MenuList/MenuItem, RadioGroup/Radio, Tablist/Tab ve Tree/TreeItem bileşenleridir.

Focusgroup’u henüz desteklemeyen tarayıcılarda [`@microsoft/focusgroup-polyfill`](https://github.com/microsoft/polyfills/tree/main/packages/focusgroup) kullanır ve bileşenler DOM’a bağlandığında otomatik olarak polyfill uygularız.

Polyfill kullanımını devre dışı bırakmak isterseniz bu bileşenlerin temel sınıflarını kullanabilirsiniz. Örneğin:

```js
import { BaseTablist } from '@iqvizyonui/web-components/tablist/base.js';

export class MyTablist extends BaseTablist {}
```
